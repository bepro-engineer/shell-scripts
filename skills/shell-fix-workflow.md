# Shell 修正ワークフロー

## 目的

このファイルは、Shell スクリプト修正時に行う作業手順を定義する。

## 前提

- 修正は必ず関数単位で行う。
- 既存コードの全面書き換えはしない。
- 修正対象ファイルだけを変更する。
- 変更対象外の関数には触れない。

## 修正前の確認

1. 対象ファイルを確認する

ユーザーが指定したファイルだけを対象にする。

2. 対象関数を確認する

修正対象の関数を明確にする。

3. 関連ルールを確認する

以下を確認する。

- CLAUDE.md
- rules 配下の該当ファイル
- 既存コードの実装方針

4. 修正方針を確認する

必要な変更だけを整理する。

## 修正時のルール

- 修正対象ファイルだけを変更する
- 差分は最小にする
- コメントだけ直す場合も、実装との整合が取れていること
- 既存セクション名・関数名を勝手に変えない
- 既存の関数名・変数名・責務を勝手に再設計しない
- 指示されていないディレクトリ名・ファイル名・構造を勝手に新規作成しない
- 不明点は推測せず停止する

## 実装時の確認観点

- 不要な Bash 固有機能の濫用は避ける
- `echo -e` より `printf` を優先する
- グローバル変数への隠れ依存を避け、必要な値は引数で受け渡す
- 冗長な `return $?` は書かない
- 未使用関数は残さず、呼び出し有無を確認して整理する
- 責務が重複している関数は、共通処理へ寄せて整理する
- 文字列比較、数値比較、空判定は境界ケースを意識する
- `set -u` 環境で落ちないように引数不足を考慮する

## 修正後の検証

修正後は必ず以下を行う。

1. 構文確認

```bash
bash -n 対象ファイル
```

2. 差分確認

```bash
git diff -- 対象ファイル
```

3. 必要最小限の動作確認

対象関数を source して直接実行し、戻り値または出力を確認する。

## 完了条件

以下を満たした場合に、修正完了とする。

- 構文確認が通っている
- 差分確認が完了している
- 必要最小限の動作確認が完了している
- 対象外ファイルに差分が出ていない
- 対象外関数を変更していない
