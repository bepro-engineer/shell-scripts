# スクリプト設計書フォーマット

## 1. スクリプト名

対象スクリプトの名称を記載する。

例:

- `CWTITOMCTL.sh`
- `serverResourceCheck.sh`

---

## 2. 機能概要

スクリプトが提供する機能を記載する。

記載観点:

- 何を行うスクリプトか
- どの対象に対して処理するか
- 引数で何を指定するか
- 対象名やサービス名をどのように決定するか
- 自動生成される値がある場合、その命名規則

---

## 3. 外装

スクリプトの実行方式・実行環境・制御方式を定義する。

| 属性 | 内容 |
|---|---|
| 記述言語 | bash / ksh / awk / ps1 / その他 |
| 起動方式 | JP1 AJS による起動 / cron による起動 / 手動起動 / その他 |
| 起動引数 | なし / あり |
| 実行時ユーザ | 特権ユーザ / 一般ユーザ / なし / その他 |
| 種別 | 単体プログラム / サブプログラム / ラッパー / リソース定義 / その他 |
| 多重起動制御方式 | ジョブネットによる制御 / ロックディレクトリ / プロセス検査 / その他 |
| 実行時言語環境 | JA_JP UTF-8 / Shift_JIS / C / その他 |
| 実行時タイムゾーン | JST-9 / GMT / その他 |
| 個別設定ファイル | なし / あり |
| 個別設定環境変数 | なし / あり |
| シグナル制御 | 制御しない / 制御する |
| ログ出力 | 標準出力・標準エラー出力のみ / リダイレクトによるログ出力 / その他 |
| ログファイル | ログファイルパス |
| 異常検知方式 | 終了コード判定 / ログトラップ / その他 |
| 終了方式 | 一連処理の終了とともに終了 / 終了条件成立による終了 / その他 |
| 終了コード | 0:正常終了 / 1:警告終了 / 2:異常終了 / その他 |
| リラン | 単純リラン可 / 単純リラン不可 |

---

## 4. 処理概要

スクリプト全体の処理の流れを記載する。

### 4.1 初期化処理

- 共通処理ファイルの読み込み
- ログ出力関数の読み込み
- 言語設定
- 戻り値初期化
- 変数初期化
- 定数定義
- サービス名や対象名の組み立てルール

### 4.2 事前処理

- `getopts` による引数取得
- 引数重複チェック
- 引数不足チェック
- 排他指定チェック
- 余剰引数チェック
- 値の妥当性チェック
- 開始ログ出力
- DEBUGログ出力
- シグナル制御
- 対象リスト作成
- 定義ファイル存在チェック
- デフォルトファイルへのフォールバック

### 4.3 主処理

- 初期戻り値設定
- 対象リスト分のループ
- コマンド種別による分岐
- 各コマンドの処理内容
- 処理後確認
- 例外時の扱い

---

## 5. 実行形式

スクリプトの実行コマンドを記載する。

記載観点:

- 基本実行形式
- 必須引数
- 任意引数
- 指定例
- Usage表示方法

---

## 6. 個別仕様

### 6.1 引数

| 引数 | オプション | 必須/任意 | 設定値 | 説明 |
|---|---|---|---|---|
| 第1引数 | `-c <実行コマンド>` | 必須 | start / stop / restart / status | 実行する処理を指定する |
| 第2引数 | `-i <対象名>` | 任意 | 任意の対象名 | 単体対象を指定する |
| 第3引数 | `-f <対象ファイル>` | 任意 | ファイルパス | 対象一覧ファイルを指定する |

記載すべき観点:

- 引数名
- オプション名
- 必須/任意
- 設定可能値
- デフォルト値
- 排他関係
- 複数指定可否
- 不正値時の動作
- Usage表示条件

### 6.2 処理仕様

コマンドまたはモードごとに処理内容を記載する。

### 6.3 出力ログ

- 開始ログ
- 終了ログ
- DEBUGログ
- INFOログ
- WARNINGログ
- ERRORログ
- 引数内容の出力
- 処理結果の出力
- 状態確認結果の出力
- 共通ログ関数の利用有無

### 6.4 終了コード

| 終了コード | 意味 |
|---:|---|
| 0 | 正常終了 |
| 1 | 警告終了 |
| 2 | 異常終了 |
| その他 | 個別仕様に記載 |

### 6.5 エラー条件

- 必須引数が未指定の場合
- 排他引数が同時指定された場合
- 対象指定が不足している場合
- command が定義外の場合
- getopts で解釈できない余剰引数が指定された場合
- 引数値が `-` で始まる場合
- 対象定義ファイルが存在しない場合
- 対象リストが0件の場合
- シグナル受信時

### 6.6 外部ファイル

| ファイル名 | 概要説明 |
|---|---|
| 共通ログ出力ファイル | 共通で使用するログ出力機能が実装されたスクリプトファイル |
| 共通関数ファイル | 共通で使用する関数が定義されたスクリプトファイル |
| 対象定義ファイル | 対象インスタンスや対象ノードを指定するための定義ファイル |

記載観点:

- ファイル名
- 配置場所
- 用途
- 必須/任意
- 存在しない場合の動作
- フォールバック有無
- ファイル形式
- 文字コード
- コメント行の扱い
- 空行の扱い
