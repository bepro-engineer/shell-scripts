# ログ出力ルール

## 目的

このファイルは、Shell スクリプトのログ出力規約を定義する。

## ログ出力ルール

- ログ出力は必ず `logOut` を使用する
- `logInfo` や `logError` などの独自略称関数は使用しない
- 形式は必ず以下とする

  logOut "INFO" "message"
  logOut "WARN" "message"
  logOut "ERROR" "message"

- 実行ファイル系 Shell Script では、前処理で必ず `startLog` を呼び出す
- `startLog` は引数解析完了後、主処理開始前に1回だけ呼び出す
- `startLog` を呼び出さずに主処理へ入らない
- 障害切り分けが目的の処理では、原因追跡に必要な情報を DEBUG レベルで記録する
- DEBUG ログには、少なくとも入力値、分岐結果、対象ファイル名、件数、主要処理の開始・終了を含める
- 障害切り分けに不要な冗長ログは出力しない
- INFO は通常運用で追跡すべき節目だけに使用する
- 開始ログ、終了ログ、ロック取得、処理継続可否などの運用上重要なイベントは INFO で記録する
- 入力引数、対象パス、算出件数、分岐内部の詳細など、調査時だけ必要な情報は DEBUG で記録する
- 標準出力で結果を返す Shell では、結果値そのものを INFO ログで重複出力しない
- 同一内容を INFO と DEBUG の両方に重複して出力しない
- INFO と DEBUG の使い分けに迷う場合は、「通常監視で必要か」「障害調査でのみ必要か」で判断する
