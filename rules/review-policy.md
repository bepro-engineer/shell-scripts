# レビュー方針

## 目的

このファイルは、レビュー時の確認観点と、レビュー段階での絶対ルールを定義する。

## レビュー観点

- shebang が `#!/bin/bash` になっているか
- 実行ファイル系 Shell Script で前処理に `startLog` があるか
- `startLog` が引数解析完了後、主処理開始前に1回だけ呼ばれているか
- Usage 表示用の関数が表示専用になっているか
- 引数不正時に当該関数内で `exitLog "${JOB_ER}"` により終了しているか
- 呼び出し元で戻り値判定方式を混在させていないか
- `-h` `--help` を実装している場合、Usage 表示後に `JOB_OK` で終了するか
- 引数なし・不正オプションで Usage 表示後に `JOB_ER` で終了するか
- 先頭行・最終行が正確に `--------------------------------------` になっているか
- Usage 表示例と1行単位で完全一致しているか
- 先頭が `-` の文字列を `printf` の format 文字列にしていないか

## レビュー段階の絶対ルール

- ユーザーが「整理」「確認」「レビュー」「設計」「問題点抽出」「修正方針」「task登録項目の整理」を依頼した段階では、コード修正を行わない
- 上記段階では、ファイルを直接編集しない
- 上記段階では、git diff が発生する変更を行わない
- 上記段階では、コード整形、リネーム、並び替え、不要行削除を含む一切の書き換えを行わない
- この段階で許可される出力は、問題点一覧、修正方針、修正対象関数、taskに分けられる修正項目一覧、想定変更点の要約のみ
- 修正後コード全文、patch、diff、直接編集は出力しない
- 実際のコード修正は、ユーザーが task 登録完了・task start 完了・feature ブランチ確認済みを明示した後にだけ行う
- 指示文に「修正はまだ行わない」「ファイルは直接編集しない」「git diff が発生する変更は行わない」が含まれている場合は、絶対に編集へ進まない
