# ファイル先頭コメントルール

## 目的

このファイルは、Shell スクリプトのファイル先頭コメントに関する規約を定義する。

## ファイル先頭コメントルール

- 実行 Shell のファイル先頭コメントは、対象ファイルに既に存在するファイルヘッダ形式を維持する
- ファイル先頭コメントでは、対象ファイルの既存ヘッダに含まれる項目を維持する
- 既存ファイルにファイル先頭コメント形式がある場合は、その形式を崩さない
- 関数コメントのテンプレートを、ファイル先頭コメントへ流用しない
- 既存ヘッダ形式が未確認の場合は、推測で標準形式を作らない

## ファイル先頭コメント内の Usage 記載ルール

- ファイル先頭コメント内に Usage がある既存スクリプトは、その形式を維持する
- Usage は対象ファイルの既存ヘッダ形式に合わせて記載する
- shebang の直後に別形式の Usage ブロックを新設しない
- `Options:` や `Example:` は、対象ファイルの既存ヘッダ形式に存在する場合は維持する
- ファイル先頭コメントの Usage 書式と、実行時の `usage()` 関数の書式を混同しない
