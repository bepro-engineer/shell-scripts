````md
# ファイル先頭コメントルール

## 目的

このファイルは、Shell スクリプトのファイル先頭コメントに関する規約を定義する。

## ファイル先頭コメントルール

- Shell スクリプトのファイル先頭コメントは、規定のヘッダーコメントテンプレートに統一する。
- 既存ファイルに異なるファイル先頭コメント形式がある場合でも、規定のヘッダーコメントテンプレートへ統一する。
- 関数コメントのテンプレートを、ファイル先頭コメントへ流用しない。
- ファイル先頭コメントへ `Usage:` / `Options:` / `Example:` / `Description:` 形式を使用しない。
- shebang は `#!/bin/bash` とする。
- 項目名と全角コロン `：` の位置を揃える。
- 項目名の位置合わせにタブを使用しない。
- 不揃いな空白で `：` の位置が崩れたヘッダーを作成しない。
- ファイル先頭コメントの修正時に、関数コメント・関数本体・実行時 `usage()` 関数を変更しない。

## 規定のヘッダーコメントテンプレート

```text
#!/bin/bash
#_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/
# スクリプト名　：dirTransfer.sh
# 概要　　　　：ディレクトリ単位の信頼性の高い転送（コピー／移動）
# 説明　　　　：
#   rsyncを使用して一時ディレクトリへ完全転送し、完了後にrenameでアトミックに入れ替える。
#   - コピー（mode=0）または移動（mode=1）に対応。
#   - 転送先に同名ディレクトリが存在する場合は削除して上書き。
#   - 親子ディレクトリ誤設定による破壊的動作を防ぐ安全設計。
#
# 引数　　　　：
#   -d <source_dir> ：転送元ディレクトリ
#   -t <target_dir> ：転送先のベースディレクトリ（直下に配置される）
#   -m <mode>       ：0=copy, 1=move（数字で指定）
#
# 戻り値　　　：0=成功, 2=異常
# 使用箇所　　：ファイル連携、バックアップ、ディレクトリ同期等
#
# 設計書　　　：なし
#_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/
# ＜変更履歴＞
# Ver. 変更管理No. 日付        更新者       変更内容
# 1.0  PR-0001
#_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/
```
````

## 項目位置合わせルール

- ヘッダー項目は下記の項目名で統一する。
  - `スクリプト名`
  - `概要`
  - `説明`
  - `引数`
  - `戻り値`
  - `使用箇所`
  - `設計書`

- 各項目の全角コロン `：` は同じ桁に揃える。
- 項目名と全角コロン `：` の間は全角スペースで調整する。
- タブを使用して位置合わせしない。
- 半角スペースとタブが混在した位置合わせをしない。

## 禁止事項

- `Usage:` / `Options:` / `Example:` / `Description:` をファイル先頭コメントの標準形式として使用しない。
- `# ------------------------------------------------------------------` をファイル先頭コメントの区切り線として使用しない。
- `# 関数名　　：` をファイル先頭コメントに使用しない。
- 関数コメントテンプレートをファイル先頭コメントへ貼り付けない。
- ファイル先頭コメントの修正と同時に、関数コメント・関数本体・実行時 `usage()` 関数を変更しない。
- shebang を `#!/bin/sh` にしない。
- 項目位置合わせにタブを使わない。
- 既存説明を勝手に要約しない。
- 既存説明を勝手に削除しない。
