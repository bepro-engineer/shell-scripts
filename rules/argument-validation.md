# 引数チェック・終了制御ルール

## 目的

このファイルは、Shell スクリプトの引数チェックと終了制御に関する規約を定義する。

## 引数チェック・終了制御ルール

- 引数チェック関数は、引数不正を検知した時点で当該関数内で終了する
- 引数チェック関数のエラー終了は必ず `exitLog "${JOB_ER}"` を使用する
- 引数チェック関数から戻り値を返して呼び出し元で判定する方式は禁止する
- 引数チェック関数について、呼び出し元で `rc` を受けて分岐する実装は禁止する
- 引数エラー時に、関数内終了方式と呼び出し元判定方式を同一スクリプト内で混在させない
- 引数なし、引数過多、不正オプションは Usage 表示後に `JOB_ER` で終了する
- help オプションを実装する場合は、Usage 表示後に `JOB_OK` で終了する
- ファイル不存在などの実体エラーは、必要なエラーログを出したうえで `JOB_ER` で終了する
- `abort` を使うことで `exitLog` を通らない実装にしない
- 引数不正を検知した時点で、当該関数内で Usage 表示と `exitLog` により終了してよい
- ただし、同一スクリプト内で引数エラー時の終了方式は統一する
